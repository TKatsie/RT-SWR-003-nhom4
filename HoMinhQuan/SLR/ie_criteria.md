# Inclusion / Exclusion Criteria

**Topic:** GPT-4 Zero-Shot for Requirement-to-Source-Code Traceability Link Recovery (vs. IR-based methods)
**Date:** July 2026

---

## Inclusion Criteria (IC)

A paper must satisfy **all** of the following criteria to be included.

| ID  | Criterion |
|-----|-----------|
| IC1 | Written in English |
| IC2 | Published in 2018 or later |
| IC3 | Peer-reviewed (journal or refereed conference proceedings) |
| IC4 | Directly addresses Traceability Link Recovery (TLR) in software engineering using LLM-based, RAG-based, or IR-based (VSM/LSI/BM25) approaches |
| IC5 | Reports empirical results with quantitative data (e.g., MAP, Recall, Precision, F1, hallucination rate) |

---

## Exclusion Criteria (EC)

A paper is excluded if it meets **any** of the following criteria.

| ID  | Criterion |
|-----|-----------|
| EC1 | Duplicate of a paper already included |
| EC2 | Full text (PDF) not accessible |
| EC3 | Poster, extended abstract, or fewer than 4 pages |
| EC4 | Does not address traceability link recovery — focuses on other tasks (e.g., requirement classification, test/test-case generation, unrelated domains such as knowledge graphs, image inspection, education) |
